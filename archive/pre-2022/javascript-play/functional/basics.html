From Scratch

Some basic ideas on functional programming.

This file includes tests for F.js (functional library) and E.js (testing library).
Console should be free of errors, see test/F.js for usage examples.
This are toy libraries for me learning functional programming, don't expect nothing impressive!
You can check Ramda, Underscore and Lodash.

Check test/F.js and src/F.js

F.identity

Returns the parameter supplied.

Equation: f(x) = x.

var one = F.identity(1);
one === 1; // true
F.curry

Translate the evaluation of a function into evaluation of a sequence of functions, each with a single argument (by partial application).

Equation: f(x, y) translated into two function calls f(x)(y) or (f(x))(y).

var add = R.curry(function(x, y) { return x + y; })
var add1 = add(1);
add1(1) === 2; // true
F.map

Higher-order function that applies the given function to each element of the list, returning a list of the results. 

F.map(function(x) { return x + 1; }, [0, 1, 2]) // [1, 2, 3]
F.transformer

Wrap any reducing function for use on reduce.

F.reduce

Higher-order function that returns a single item by iterating through the list and calling the iterator function with the accumulator and the current value from the list, and then passing the result to the next call. 

F.reduce(function(x, y) { return x + y; }, 0, [1, 1, 1]) // 3
F.filter

Higher-order function that processes a list to produce a new list containing exactly the elements of the original list for which the predicate function returns true. 

F.filter(function(x) { return x > 1; }, [3, 2, 1]) // [3, 2]
