Pure

A pure function is one that given the same input, will always return the same output, without observable side effect.
This means that you can replace a pure function with the returned value without causing problems. This is known as referential transparency.
slice is a pure function because it returns the same output per input every time, splice is not pure because it returns a different value.
A pure function does not depend on any external state.
Pure functions allow interesting things like easier testing, parallelization, memoization and laziness. Also pure functions tend to be self-documented and simpler.

Side Effects

A function or expression is said to have a side effect, in addition to returning a value, if it also modifies some state or has an observable interaction with calling functions or the outside world.
Temporal side effects, like Sleep(500) or the time an iteration takes to complete, are usually ignored when discussing side effects.
Examples of side effects: changes on file system, inserting or updating a database record, http call, printing to screen, mutation...
Side effects can be controlled with things like functors and monads.
