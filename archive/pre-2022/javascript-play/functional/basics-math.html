Associative

When an expression containing two or more occurrences in a row of the same associative operator, the order in which the operations are performed does not matter as long as the sequence of the operands is not changed.
Example: (2 + 3) + 4 = 2 + (3 + 4)
From wikipedia: Associative Property 

E.strictEqual(add(add(a, b), c), add(a, add(b, c)));

Commutative

An operation is commutative if changing the order of the operands does not change the result.
Not all operations are commutative, multiplication and addition are for real numbers, but not division, subtraction and concatenation.
More information: Commutative Property. 

E.strictEqual(add(a, b), add(b, a));

Identity

The identity element, shortened as identity, is a special type of element from a set, it leaves other elements unchanged when combined with them.
For example in real numbers 0 (zero) is the identity element with addition. And 1 (one) with multiplication.
More information: Identity Element 

E.strictEqual(add(a, 0), a);

Distributive

Distribution refers to two valid rules of replacement.
Example: 2 * (1 + 3) = (2 * 1) + (2 * 3)
More information: Distributive Property 

E.strictEqual(multiply(a, add(b, c)), add(multiply(a, b), multiply(a, c)));
